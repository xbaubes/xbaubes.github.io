Eines com SheetDB són útils perquè converteixen un full de càlcul en una API REST, és a dir, en un servei web amb el qual pots comunicar-te per llegir, afegir, editar o eliminar dades utilitzant crides HTTP (GET, POST, PUT, DELETE).

En lloc d’haver de:

  • Configurar un servidor propi,
  • Escriure codi backend,
  • O gestionar una base de dades tradicional,

...pots aprofitar Google Sheets com a magatzem de dades i fer-lo accessible de manera estructurada i estàndard.

Utilitat

Simplicitat

  • No cal saber backend ni SQL per tenir un sistema de dades compartit.
  • L'usuari pot modificar la les dades directament al full.

Integració fàcil

  • Qualsevol aplicació web, app mòbil o script pot consumir o enviar dades a través de l’API.
  • Funciona amb JavaScript al navegador, Python, PHP, etc.

Col·laboració

  • El full es pot editar per múltiples persones alhora i les actualitzacions són immediates a l’API.

Versatilitat

  • Serveix tant per projectes petits (formularis, mapes, llistats) com per prototips d’aplicacions més grans.

Funcionament

  1. Crees el full de càlcul amb les columnes (capçaleres) que seran els camps de dades.
  2. El connectes a SheetDB i obtens una URL d’API.
  3. Pots fer:
    • GET: obtenir dades (https://sheetdb.io/api/v1/ID o /search?...)
    • POST: afegir noves files
    • PUT/PATCH: modificar files existents
    • DELETE: esborrar files
  4. Tot queda sincronitzat amb el full en temps real.

    • Si la resposta és correcta, la converteix a JSON.
